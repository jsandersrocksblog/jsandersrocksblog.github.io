<h2 style="margin: 10pt 0in 0pt">
  <font color="#4f81bd" size="4" face="Cambria">Overview</font>
</h2>

<p style="margin: 0in 0in 0pt" class="MsoNormal">
  <font size="3" face="Calibri">You can solve issues encountered when using the HTTP protocol by applying the principles in this article.<span style="mso-spacerun: yes">&nbsp; </span>By simplifying the issue into a small reproducible problem you can effectively troubleshoot and resolve most problems encountered in code that uses the HTTP protocol.<span style="mso-spacerun: yes">&nbsp; </span>Crucial steps to troubleshooting code are: Gather Information, Simplify the Problem and Detailed Troubleshooting.</font>
</p>

<h2 style="margin: 10pt 0in 0pt">
  <font color="#4f81bd" size="4" face="Cambria">Technologies Covered</font>
</h2>

<p style="margin: 0in 0in 0pt" class="MsoNormal">
  <span style="mso-bidi-font-family: arial"><font size="3"><font face="Calibri">WinHttp, WinInet, Internet Explorer, XMLHTTP (MSXML), ServerXMLHTTP, System.Net classes such as WebClient, HttpWebRequest, SOAP classes, WebServices, System.Xml classes such as XmlUrlResolver, SoapHttpClientProtocol. <o:p></o:p></font></font></span>
</p>

<p style="margin: 0in 0in 0pt" class="MsoNormal">
  <span style="mso-bidi-font-family: arial"><o:p><font size="3" face="Calibri">&nbsp;</font></o:p></span>
</p>

<h2 style="margin: 10pt 0in 0pt">
  <font color="#4f81bd" size="4" face="Cambria">Error Messages Encountered (so search will find this document)</font>
</h2>

<p style="margin: 0in 0in 0pt" class="MsoNormal">
  <font size="3"><font face="Calibri"><span style="mso-bidi-font-family: arial">The operation timed out, 80072EE2, -2147954402, ERROR_WINHTTP_TIMEOUT, ERROR_WINHTTP_INVALID_SERVER_RESPONSE, The server name or address could not be resolved, 0x80072EE7, </span>-2147012889, <span style="mso-bidi-font-family: arial">ERROR_WINHTTP_NAME_NOT_RESOLVED,</span> <span style="mso-bidi-font-family: arial">0x80072EFD<o:p></o:p></span></font></font>
</p>

<p style="margin: 0in 0in 0pt" class="MsoNormal">
  <font size="3"><font face="Calibri">-2147012867, A connection with the server could not be established, <span style="mso-bidi-font-family: arial">-2146697208 (800c0008), The download of the specified resource has failed, Cannot connect to Remote server, Access is Denied, System error: -2146697191, 401, 500, 407, Server refused connection, Http protocol violation, Unauthorized, Cannot find server.<o:p></o:p></span></font></font>
</p>

<p style="margin: 0in 0in 0pt" class="MsoNormal">
  <span style="mso-bidi-font-family: arial"><o:p><font size="3" face="Calibri">&nbsp;</font></o:p></span>
</p>

<h2 style="margin: 10pt 0in 0pt">
  <font color="#4f81bd" size="4" face="Cambria">Step 1.<span style="mso-spacerun: yes">&nbsp; </span>Gather Information</font>
</h2>

<p style="margin: 0in 0in 0pt" class="MsoNormal">
  <span style="mso-bidi-font-family: arial"><font size="3"><font face="Calibri">You may solve the issue or get close to the root cause quickly by gathering information about the problem.<span style="mso-spacerun: yes">&nbsp; </span>Then you can continue to the <b style="mso-bidi-font-weight: normal">Simplify the Problem</b> section of this document to further isolate the issue and determine where the problem is occurring. Do this only after Gathering the information listed below.<span style="mso-spacerun: yes">&nbsp; </span><span style="mso-bidi-font-weight: bold"><o:p></o:p></span></font></font></span>
</p>

<p style="margin: 0in 0in 0pt 0.25in" class="MsoNormal">
  <span style="mso-bidi-font-family: arial"><o:p><font size="3" face="Calibri">&nbsp;</font></o:p></span>
</p>

<p style="text-indent: -0.25in; margin: 0in 0in 0pt 0.75in; mso-list: l0 level1 lfo1" class="MsoListParagraph">
  <span style="mso-bidi-font-family: calibri; mso-bidi-theme-font: minor-latin"><span style="mso-list: ignore"><font size="3" face="Calibri">1.</font><span style="font: 7pt 'Times New Roman'">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span></span></span><span style="mso-bidi-font-family: arial"><font size="3"><font face="Calibri">What HTTP Verb are you using (EX: GET, POST, PUT)?<o:p></o:p></font></font></span>
</p>

<p style="text-indent: -0.25in; margin: 0in 0in 0pt 0.75in; mso-list: l0 level1 lfo1" class="MsoListParagraph">
  <span style="mso-bidi-font-family: calibri; mso-bidi-theme-font: minor-latin"><span style="mso-list: ignore"><font size="3" face="Calibri">2.</font><span style="font: 7pt 'Times New Roman'">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span></span></span><span style="mso-bidi-font-family: arial"><font size="3"><font face="Calibri">What is the destination URL?<o:p></o:p></font></font></span>
</p>

<p style="text-indent: -0.25in; margin: 0in 0in 0pt 0.75in; mso-list: l0 level1 lfo1" class="MsoListParagraph">
  <span style="mso-bidi-font-family: calibri; mso-bidi-theme-font: minor-latin"><span style="mso-list: ignore"><font size="3" face="Calibri">3.</font><span style="font: 7pt 'Times New Roman'">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span></span></span><span style="mso-bidi-font-family: arial"><font size="3"><font face="Calibri">Did this ever work? What changed if it did work previously (moved to production server, applied Service pack or Anti Virus update)?<o:p></o:p></font></font></span>
</p>

</p>

<p style="text-indent: -0.25in; margin: 0in 0in 0pt 0.75in; mso-list: l0 level1 lfo1" class="MsoListParagraph">
  <span style="mso-bidi-font-family: calibri; mso-bidi-theme-font: minor-latin"><span style="mso-list: ignore"><font size="3" face="Calibri">6.</font><span style="font: 7pt 'Times New Roman'">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span></span></span><span style="mso-bidi-font-family: arial"><font size="3"><font face="Calibri">Can you reach ANY URL from the computer (Perhaps DNS is not set, ports blocked, or you need a proxy setting)?<o:p></o:p></font></font></span>

<h2 style="margin: 10pt 0in 0pt">
  <font color="#4f81bd" size="4" face="Cambria">Step 2. Simplify the Problem</font>
</h2>

<p style="margin: 0in 0in 0pt" class="MsoNormal">
  <o:p><font size="3" face="Calibri">&nbsp;</font></o:p>
</p>

<p style="margin: 0in 0in 0pt" class="MsoNormal">
  <span style="mso-bidi-font-family: arial"><font size="3"><font face="Calibri">Armed with the answers from Step 1, reproduce the problem with the simplest code possible.<span style="mso-spacerun: yes">&nbsp;&nbsp; </span>In the process of simplifying the issue you may discover the issue is not in the technology you are using but a problem being obscured by your program’s logic or system configuration. <span style="mso-spacerun: yes">&nbsp;</span>For example, In one case I assisted in, numerous troubleshooting actions had been taken in the ASP pages to determine what was going wrong.<span style="mso-spacerun: yes">&nbsp; </span>When I was brought in to assist, I simply asked, “Can you hit the target URL in a Browser”?<span style="mso-spacerun: yes">&nbsp; </span>The answer (“no”) instantly revealed to all of us that the issue was the DNS server setting on that machine and not the code.<o:p></o:p></font></font></span>
</p>

<p style="margin: 0in 0in 0pt 0.25in" class="MsoNormal">
  <span style="mso-bidi-font-family: arial"><o:p><font size="3" face="Calibri">&nbsp;</font></o:p></span>
</p>

<p style="margin: 0in 0in 0pt" class="MsoNormal">
  <span style="mso-bidi-font-family: arial"><font size="3"><font face="Calibri">How to simplify the issue:<o:p></o:p></font></font></span>
</p>

<p style="margin: 0in 0in 0pt" class="MsoNormal">
  <span style="mso-bidi-font-family: arial"><o:p><font size="3" face="Calibri">&nbsp;</font></o:p></span>
</p>

<p style="text-indent: -0.25in; margin: 0in 0in 0pt 0.75in; mso-list: l1 level1 lfo2" class="MsoListParagraph">
  <span style="mso-bidi-font-family: calibri; mso-bidi-theme-font: minor-latin"><span style="mso-list: ignore"><font size="3" face="Calibri">1.</font><span style="font: 7pt 'Times New Roman'">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span></span></span><span style="mso-bidi-font-family: arial"><font size="3"><font face="Calibri">Duplicate the issue with a technology that is easier to troubleshoot such as the Web Browser or System.Net 2.0 which has the ability to log the HTTPS traffic.<o:p></o:p></font></font></span>
</p>

<p style="text-indent: -0.25in; margin: 0in 0in 0pt 0.75in; mso-list: l1 level1 lfo2" class="MsoListParagraph">
  <span style="mso-bidi-font-family: calibri; mso-bidi-theme-font: minor-latin"><span style="mso-list: ignore"><font size="3" face="Calibri">2.</font><span style="font: 7pt 'Times New Roman'">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span></span></span><span style="mso-bidi-font-family: arial"><font size="3"><font face="Calibri">If the problem occurs with HTTPS, see if you can reproduce with HTTP as well then troubleshoot HTTP traffic instead of HTTPS traffic (which is securely encoded).<span style="mso-spacerun: yes">&nbsp; </span>If you cannot duplicate the issue with HTTP then the problem may lie with your HTTPS configuration.<o:p></o:p></font></font></span>
</p>

<p style="text-indent: -0.25in; margin: 0in 0in 0pt 0.75in; mso-list: l1 level1 lfo2" class="MsoListParagraph">
  <span style="mso-bidi-font-family: calibri; mso-bidi-theme-font: minor-latin"><span style="mso-list: ignore"><font size="3" face="Calibri">3.</font><span style="font: 7pt 'Times New Roman'">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span></span></span><span style="mso-bidi-font-family: arial"><font size="3"><font face="Calibri">If this is a service based application, such as ASP or ASP.NET, move the code out of the Service and test.<span style="mso-spacerun: yes">&nbsp; </span>If it works, then the issue is not with the technology you are using.<span style="mso-spacerun: yes">&nbsp; </span>The issue is how you have your service configured or the context it is running in.<o:p></o:p></font></font></span>
</p>

<p style="text-indent: -0.25in; margin: 0in 0in 0pt 0.75in; mso-list: l1 level1 lfo2" class="MsoListParagraph">
  <span style="mso-bidi-font-family: calibri; mso-bidi-theme-font: minor-latin"><span style="mso-list: ignore"><font size="3" face="Calibri">4.</font><span style="font: 7pt 'Times New Roman'">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span></span></span><span style="mso-bidi-font-family: arial"><font size="3"><font face="Calibri">Try simple code from script or a command line application to see if that succeeds.<span style="mso-spacerun: yes">&nbsp; </span>Then use that same simple code in your application and test.<span style="mso-spacerun: yes">&nbsp; </span>If it fails in either case you have a much smaller problem to deal with.<o:p></o:p></font></font></span>
</p>

<p style="text-indent: -0.25in; margin: 0in 0in 0pt 0.75in; mso-list: l1 level1 lfo2" class="MsoListParagraph">
  <span style="mso-bidi-font-family: calibri; mso-bidi-theme-font: minor-latin"><span style="mso-list: ignore"><font size="3" face="Calibri">5.</font><span style="font: 7pt 'Times New Roman'">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span></span></span><span style="mso-bidi-font-family: arial"><font size="3"><font face="Calibri">Simplify the code by executing the simplest possible HTTP call in a command line script or console application.<span style="mso-spacerun: yes">&nbsp; </span>Start with using a HEAD or GET verb to a resource (perhaps an image or static file on the server) and add complexity until you get the failure.<o:p></o:p></font></font></span>
</p>

<p style="margin: 0in 0in 0pt 0.25in" class="MsoNormal">
  <span style="mso-bidi-font-family: arial"><o:p><font size="3" face="Calibri">&nbsp;</font></o:p></span>
</p>

  <h2 style="margin: 10pt 0in 0pt">
    <font color="#4f81bd" size="4" face="Cambria">Step 3. Detailed Troubleshooting</font>
  </h2>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <span style="color: #1f497d; mso-themecolor: dark2"><o:p><font size="3" face="Calibri">&nbsp;</font></o:p></span>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <font size="3" face="Calibri">DO NOT troubleshoot until you simplify the problem.<span style="mso-spacerun: yes">&nbsp; </span>Get the problem isolated first to the simplest possible reproduction of the problem (following Step 2) and then proceed to this step.<span style="mso-spacerun: yes">&nbsp; </span>For example: If you are having trouble hitting a WebService from ASP.NET and can reproduce the issue with a simple command line application written in .NET, you will be able to troubleshoot this much easier.</font>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <o:p><font size="3" face="Calibri">&nbsp;</font></o:p>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <font size="3" face="Calibri">Volumes of material could be written on this subject.<span style="mso-spacerun: yes">&nbsp; </span>This document will not give you detailed troubleshooting steps but simply ideas to get you started.</font>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <o:p><font size="3" face="Calibri">&nbsp;</font></o:p>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <b style="mso-bidi-font-weight: normal"><font size="3"><font face="Calibri">Search<o:p></o:p></font></font></b>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <font size="3" face="Calibri">Many times after you have simplified the issue, you can search MSDN and use </font><a href="http://search.live.com/" mce_href="http://search.live.com/"><span style="mso-bidi-font-weight: bold"><font size="3" face="Calibri">http://Search.Live.Com</font></span></a><font size="3" face="Calibri"> to find the issue and solution to your particular situation.</font>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <o:p><font size="3" face="Calibri">&nbsp;</font></o:p>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <b style="mso-bidi-font-weight: normal"><font size="3"><font face="Calibri">Compare<o:p></o:p></font></font></b>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <font size="3" face="Calibri">In almost every case you can gather data of a successful HTTP request and compare it to the failed case.<span style="mso-spacerun: yes">&nbsp; </span>From this you can see what the difference is between the successful and failed request and correct the issue.<span style="mso-spacerun: yes">&nbsp; </span>The problem may be missing or incorrect headers, data or authorization.</font>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <o:p><font size="3" face="Calibri">&nbsp;</font></o:p>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <font size="3" face="Calibri">Two useful tools for gathering HTTP traffic are ‘Network Monitor’ and ‘Fiddler’.<span style="mso-spacerun: yes">&nbsp; </span>‘Fiddler’ has the advantage of being able to execute requests you can construct yourself, the ability to see HTTPS traffic using a ‘man in the middle’ approach and will detect HTTP Protocol violations that the .NET classes will report as WebExceptions.<span style="mso-spacerun: yes">&nbsp; </span>‘Network Monitor’ has the advantage of showing you not only the HTTP traffic but the underlying TCP transport traffic.<span style="mso-spacerun: yes">&nbsp; </span>This makes it a useful tool when the issue is the transport (TCP) instead of the protocol (HTTP).</font>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <o:p><font size="3" face="Calibri">&nbsp;</font></o:p>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <font size="3" face="Calibri">Both tools have excellent help documentation built into them.<span style="mso-spacerun: yes">&nbsp; </span>Familiarize yourself with these help files and practice capturing data with them.</font>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <span style="color: red"><o:p><font size="3" face="Calibri">&nbsp;</font></o:p></span>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <b style="mso-bidi-font-weight: normal"><font size="3"><font face="Calibri">Use Logging<o:p></o:p></font></font></b>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <font size="3" face="Calibri">Another useful tool is logging.<span style="mso-spacerun: yes">&nbsp; </span>WinInet and WinHttp as well as the System.Net based technologies have the ability to generate logs.<span style="mso-spacerun: yes">&nbsp; </span>Often these logs will help you find problems and see exactly what is happening internally.</font>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <o:p><font size="3" face="Calibri">&nbsp;</font></o:p>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <font size="3" face="Calibri">WinHttp logging (also used by ServerXMLHttp): </font><a href="http://msdn2.microsoft.com/en-us/library/aa384119(VS.85).aspx" mce_href="http://msdn2.microsoft.com/en-us/library/aa384119(VS.85).aspx"><font color="#0000ff" size="3" face="Calibri">http://msdn2.microsoft.com/en-us/library/aa384119(VS.85).aspx</font></a><font size="3"><font face="Calibri"><span style="mso-spacerun: yes">&nbsp; </span>Vista and Above use netsh: </font></font><a href="http://msdn2.microsoft.com/en-us/library/bb648687.aspx" mce_href="http://msdn2.microsoft.com/en-us/library/bb648687.aspx"><font color="#0000ff" size="3" face="Calibri">http://msdn2.microsoft.com/en-us/library/bb648687.aspx</font></a>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <o:p><font size="3" face="Calibri">&nbsp;</font></o:p>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <font size="3" face="Calibri">WinInet logging (also used by XMLHTTP): </font><a href="http://support.microsoft.com/kb/884931" mce_href="http://support.microsoft.com/kb/884931"><font size="3" face="Calibri">http://support.microsoft.com/kb/884931</font></a>&nbsp;
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <o:p><font size="3" face="Calibri">WinInet logging in Vista: <a href="http://blogs.msdn.com/wndp/archive/2006/03/28/wininet-etw-tracing-support-in-windows-vista.aspx">http://blogs.msdn.com/wndp/archive/2006/03/28/wininet-etw-tracing-support-in-windows-vista.aspx</a></font></o:p>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <o:p><font size="3" face="
Calibri"></font></o:p>&nbsp;
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <font size="3"><font face="Calibri">System.Net logging:<span style="mso-spacerun: yes">&nbsp; </span></font></font><a href="http://msdn2.microsoft.com/en-us/library/ty48b824(VS.80).aspx" mce_href="http://msdn2.microsoft.com/en-us/library/ty48b824(VS.80).aspx"><font color="#0000ff" size="3" face="Calibri">http://msdn2.microsoft.com/en-us/library/ty48b824(VS.80).aspx</font></a>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <o:p><font size="3" face="Calibri">&nbsp;</font></o:p>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <b style="mso-bidi-font-weight: normal"><font size="3"><font face="Calibri">See what is happening at the Protocol level<o:p></o:p></font></font></b>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <font size="3" face="Calibri">Using Fiddler or Netmon, see if the server is responding.<span style="mso-spacerun: yes">&nbsp; </span>If it is, what is the HTTP Status code?<span style="mso-spacerun: yes">&nbsp; </span>A 407 means the Proxy needs credentials for authorization.<span style="mso-spacerun: yes">&nbsp; </span>Make sure you are sending those credentials after the 407 comes back.<span style="mso-spacerun: yes">&nbsp; </span>Similarly ensure your application responds to Server Authentication requests (HTTP Status Code 401).<span style="mso-spacerun: yes">&nbsp; </span>If the server is responding 500 series of status codes, then the issue is on the server side.<span style="mso-spacerun: yes">&nbsp; </span>Are you sending the correct data to the server?<span style="mso-spacerun: yes">&nbsp; </span>Does the server work for the same exact query from different clients?<span style="mso-spacerun: yes">&nbsp; </span>If not then the server programmer needs to resolve the issue.</font>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <font size="3" face="Calibri">Capture traffic on the client and server to see if anything is altering or delaying the traffic between the two.</font>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <o:p><font size="3" face="Calibri">&nbsp;</font></o:p>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <b style="mso-bidi-font-weight: normal"><font size="3"><font face="Calibri">Sporadic problems<o:p></o:p></font></font></b>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <font size="3" face="Calibri">These are the most difficult to narrow down and a detailed discussion of this could fill a book (or website).<span style="mso-spacerun: yes">&nbsp; </span>Many times the issue is a resource issue on the client or server.<span style="mso-spacerun: yes">&nbsp; </span>You need to isolate the issue if possible to a simple repro at the time of failure.<span style="mso-spacerun: yes">&nbsp; </span>You can try getting trapping 1<sup>st</sup> chance WebExceptions if using .NET or a .NET log to see what is happening.<span style="mso-spacerun: yes">&nbsp; </span>Logging and network traces will be your best bet to solve most of these issues.<span style="mso-spacerun: yes">&nbsp; </span>In the ASP.NET case, you may have to get a manual dump of the w3wp.exe process and engage Microsoft support for assistance.</font>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <o:p><font size="3" face="Calibri">&nbsp;</font></o:p>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <font size="3" face="Calibri">Perfmon counters may show you what is going on as well.<span style="mso-spacerun: yes">&nbsp; </span>Ultimately you may have to get a Debug Dump and engage Microsoft Support if you cannot isolate the issue yourself.<span style="mso-spacerun: yes">&nbsp; </span>When the problem occurs you can use DebugDiagnostics to get a couple of hang dumps and then analyze these dumps with that same tool and it will often suggest what the issue is and how to fix it.</font>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <o:p><font size="3" face="Calibri">&nbsp;</font></o:p>
  </p>
  
  <h2 style="margin: 10pt 0in 0pt">
    <font color="#4f81bd" size="4" face="Cambria">Other troubleshooting hints:</font>
  </h2><p style="margin: 0in 0in 0pt" class="MsoNormal" mce_keep="true">&nbsp;</p> 
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <font size="3" face="Calibri">Test the possibility of the Network causing the issue by putting the client and server on the same box.&nbsp; If the problem goes away, then you may have an issue with the Network or network devices. </font>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <font size="3" face="Calibri"></font>&nbsp;
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <font size="3" face="Calibri">Temporarily Eliminate load balancing and access a particular server directly so that you can determine if the load balancer is a factor and allow you to monitor a particular server while troubleshooting.</font>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <font size="3" face="Calibri"></font>&nbsp;
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <font size="3" face="Calibri">Does the request look exactly the same as it left the client when it gets to the server?<span style="mso-spacerun: yes">&nbsp; </span>Use<span style="mso-spacerun: yes">&nbsp; </span>Netmon<span style="mso-spacerun: yes">&nbsp; </span>to determine this.<span style="mso-spacerun: yes">&nbsp; </span>It is possible that a Network device is altering packets in transit.</font>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <o:p><font size="3" face="Calibri">&nbsp;</font></o:p>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <font size="3" face="Calibri">Can any other client machine make the same request successfully?<span style="mso-spacerun: yes">&nbsp; </span>It must be a configuration issue on the problem machine if other clients can get there.</font>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <o:p><font size="3" face="Calibri">&nbsp;</font></o:p>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <font size="3" face="Calibri">Is the problem only Http or Https traffic?<span style="mso-spacerun: yes">&nbsp; </span>Https problems may be certificate related.<span style="mso-spacerun: yes">&nbsp; </span>Ensure client certificate is installed in the Local Machine store for ASP/ASP.NET apps (see link below &#8220;How to call a Web service by using a client certificate for authentication in an ASP.NET Web application&#8221;) or current user store for non service based apps if required.<span style="mso-spacerun: yes">&nbsp; </span>Ensure the certificate chain is trusted by viewing the certificate chain in Internet Explorer. </font>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <o:p><font size="3" face="Calibri">&nbsp;</font></o:p>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <font size="3" face="Calibri">Does the request leave the box?<span style="mso-spacerun: yes">&nbsp; </span>Use Netmon or Fiddler or tracing to determine this.<span style="mso-spacerun: yes">&nbsp; </span>AntiVirus or firewall applications are suspect so temporarily disable them.</font>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <o:p><font size="3" face="Calibri">&nbsp;</font></o:p>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <font size="3" face="Calibri">Does any response come back from the server?<span style="mso-spacerun: yes">&nbsp; </span>Use Netmon or Fiddler or tracing to determine this.<span style="mso-spacerun: yes">&nbsp; </span>Perhaps the server is not responding due to server errors.<span style="mso-spacerun: yes">&nbsp; </span>Inspect server logs and WebService logs.</font>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <o:p><font size="3" face="Calibri">&nbsp;</font></o:p>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <b style="mso-bidi-font-weight: normal"><font size="3"><font face="Calibri">Some issues I have encountered:<o:p></o:p></font></font></b>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <o:p><font size="3" face="Calibri">&nbsp;</font></o:p>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <font size="3" face="Calibri">Not able to reach URL even with simplified code using different technologies. DNS or Proxy issues on network or on that particular machin<br /> e could be the cause.<span style="mso-spacerun: yes">&nbsp; </span>Try a different machine to confirm or narrow this down</font>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <o:p><font size="3" face="Calibri">&nbsp;</font></o:p>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <font size="3" face="Calibri">Can reach URL with one Technology but not another:<span style="mso-spacerun: yes">&nbsp; </span>Common issues are: <span style="mso-spacerun: yes">&nbsp;</span>Authorization not done correctly, Proxy settings, Firewall settings, or Antivirus is blocking the outgoing call.<span style="mso-spacerun: yes">&nbsp; </span>Compare successful vs. failed cases to see what is happening.</font>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <o:p><font size="3" face="Calibri">&nbsp;</font></o:p>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <font size="3" face="Calibri">If you can reach that URL and successfully retrieve data using one technology but not another, then determine:</font>
  </p>
  
  <p style="text-indent: -0.25in; margin: 0in 0in 0pt 0.5in; mso-list: l2 level1 lfo3" class="MsoListParagraph">
    <span style="font-family: symbol; mso-bidi-font-family: symbol; mso-fareast-font-family: symbol"><span style="mso-list: ignore"><font size="3">·</font><span style="font: 7pt 'Times New Roman'">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span></span></span><font size="3" face="Calibri">Does the HTTP traffic in both cases look the same?</font>
  </p>
  
  <p style="text-indent: -0.25in; margin: 0in 0in 0pt 0.5in; mso-list: l2 level1 lfo3" class="MsoListParagraph">
    <span style="font-family: symbol; mso-bidi-font-family: symbol; mso-fareast-font-family: symbol"><span style="mso-list: ignore"><font size="3">·</font><span style="font: 7pt 'Times New Roman'">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span></span></span><font size="3" face="Calibri">Does the time for the response differ from one client to another?</font>
  </p>
  
  <p style="text-indent: -0.25in; margin: 0in 0in 0pt 0.5in; mso-list: l2 level1 lfo3" class="MsoListParagraph">
    <span style="font-family: symbol; mso-bidi-font-family: symbol; mso-fareast-font-family: symbol"><span style="mso-list: ignore"><font size="3">·</font><span style="font: 7pt 'Times New Roman'">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span></span></span><font size="3" face="Calibri">Use various tools to determine the above to include; logging, Netmon 3.2, Fiddler, Simultaneous Netmon traces on client and target machines.<span style="mso-spacerun: yes">&nbsp; </span>Fiddler can be configured to see the HTTPS traffic as clear text if it is an HTTPS only problem.</font>
  </p>
  
  <p style="text-indent: -0.25in; margin: 0in 0in 0pt 0.5in; mso-list: l2 level1 lfo3" class="MsoListParagraph">
    <span style="font-family: symbol; mso-bidi-font-family: symbol; mso-fareast-font-family: symbol"><span style="mso-list: ignore"><font size="3">·</font><span style="font: 7pt 'Times New Roman'">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; </span></span></span><font size="3" face="Calibri">AntiVirus and Firewall programs can block the traffic and do this differently per process.&nbsp; Test by turning them off.</font>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <o:p><font size="3" face="Calibri">&nbsp;</font></o:p>
  </p>
  
  <p style="margin: 0in 0in 0pt" class="MsoNormal">
    <font size="3" face="Calibri">Sometimes you will see a 401 response in ASP.NET application and not in console app with same code.<span style="mso-spacerun: yes">&nbsp; </span>The issue commonly is that the credentials passed by default are obtained from the security context that the program was running in and ASP.NET was not using the logged on credentials (search ASP.NET impersonation on MSDN).</font>
  </p>
